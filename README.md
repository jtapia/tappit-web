# TappitX

Tap-to-click for Apple Magic Mouse. A lightweight macOS menu bar utility that converts surface taps into real mouse clicks — with pressure-sensitive right-clicks, configurable zones, haptic feedback, and per-app rules.

This repository is the **marketing website + supporting Cloudflare Workers**. The macOS app source lives in a separate private repo (`jtapia/TappitX`).

**Website**: Built with Next.js 16, Tailwind CSS 4, and Framer Motion. Static export, served from Cloudflare.

## Features

**Core**: Tap-to-click (single & double), zone-based right-click, configurable sensitivity, battery monitoring, start on login.

**Pro**: Pressure-based right-click, middle-click center zone, per-app right-click rules, haptic & sound feedback, visual tap overlay, scroll customization, palm rejection.

## Tech Stack

- **Next.js 16** with App Router and static export (`output: "export"`)
- **React 19** + **TypeScript 5**
- **Tailwind CSS 4** with CSS custom properties for dark/light theming
- **Framer Motion 12** for scroll-triggered animations
- **Stripe Payment Links** for checkout (no backend required)
- **pnpm** as the package manager (workspace with the `license-issuer` worker)

## Getting Started

This repo uses **pnpm** (see `packageManager` in `package.json`). Install pnpm with `corepack enable` if you don't have it.

```bash
pnpm install
pnpm dev          # http://localhost:3000
pnpm build        # Static output in /out
pnpm lint
```

> **Note**: pnpm v10+ blocks dependency build scripts by default. Build-script decisions live in `pnpm-workspace.yaml` under `allowBuilds` (`esbuild`, `unrs-resolver`, `workerd` are enabled; `sharp` is intentionally skipped — the site uses static export with `images: { unoptimized: true }`, so image optimization never runs). Do not move these back to `package.json` or the build breaks.

## Deployment

The site is a static export deployed to Cloudflare. The two Workers (`worker/` and `workers/license-issuer/`) deploy separately.

### Website

```bash
pnpm build && npx wrangler pages deploy out
```

Or connect the repo in the Cloudflare dashboard with:
- Build command: `pnpm build`
- Output directory: `out`

### Main site Worker (`gettappit.com`)

Serves static assets and the `/api/cta` endpoint that drives the buy buttons.

```bash
npx wrangler deploy        # uses ./wrangler.toml
```

### License-issuer Worker (`license.gettappit.com`)

```bash
cd workers/license-issuer
wrangler deploy
```

## Releasing the macOS app (DMG + auto-update)

The app binary is **built, signed, and notarized in the app repo** (`jtapia/TappitX`), then **hosted publicly** and **pointed at from this repo**. Because the source repo is private, the DMG must live in the public **`jtapia/TappitX-releases`** repo (or Cloudflare R2).

For a new version (example: `1.0.1`):

1. **Build & sign** in the app repo. `scripts/release.sh` produces the notarized DMG and `build/appcast-snippet.xml` (which contains the Sparkle `sparkle:edSignature` and `length` — the EdDSA signature of that exact DMG, generated by Sparkle's `sign_update`).

2. **Publish the DMG** as a GitHub Release asset in the public repo (the filename must match `NEXT_PUBLIC_DOWNLOAD_URL`):

   ```bash
   gh release create v1.0.1 ./build/TappitX-1.0.1.dmg \
     --repo jtapia/TappitX-releases --title "TappitX 1.0.1"
   ```

   Resulting download URL:
   `https://github.com/jtapia/TappitX-releases/releases/download/v1.0.1/TappitX-1.0.1.dmg`

   > If the releases repo is brand new (no commits), `gh release create` fails with `Invalid target_commitish`. Seed it with one commit first (e.g. add a `README.md`) so it has a default branch.

3. **Point this repo at the new version:**
   - `.env.production` → bump `NEXT_PUBLIC_DOWNLOAD_URL` (also `wrangler.toml [vars]` if deploying via the Worker).
   - `public/appcast.xml` → add a new `<item>` (most recent first), copying `edSignature` + `length` from the app repo's `build/appcast-snippet.xml`. Re-add `<sparkle:releaseNotesLink>` once a `release-notes/<version>.html` page exists.

4. **Deploy the site** so the updated download link and `appcast.xml` go live. Sparkle in-app auto-update reads `https://gettappit.com/appcast.xml`.

Verify the feed after deploy:

```bash
curl -s https://gettappit.com/appcast.xml | xmllint --format -
```

> ⚠️ The `edSignature` is only valid if the **shipped app build** embeds the `SUPublicEDKey` matching the private key `release.sh` signs with. If you rotate the signing key, rebuild the app with the new public key or Sparkle will reject the update as improperly signed.

## Stripe Setup

1. Create a Payment Link at [dashboard.stripe.com/payment-links](https://dashboard.stripe.com/payment-links).
2. Set `PURCHASE_URL` in `wrangler.toml`:
   ```toml
   [vars]
   PURCHASE_URL = "https://buy.stripe.com/your_link_here"
   ```
3. Deploy: `wrangler deploy`.

The buy buttons fetch `/api/cta` from the Worker at runtime (via the `useCtaHref()` hook in `src/hooks/useCta.ts`), so the link updates without a rebuild. If the Worker is unreachable, the client falls back to the `/#pricing` anchor. (The Hero's primary button links straight to `/#pricing` so visitors see pricing before checkout.)

## License Issuer Worker

A separate Cloudflare Worker at `license.gettappit.com` handles post-purchase license delivery and activation. Source lives in `workers/license-issuer/`.

**Endpoints:**

| Route | Description |
|-------|-------------|
| `POST /webhook` | Stripe webhook — creates and emails a license on successful payment |
| `GET /session/:id` | Returns masked purchase confirmation for the `/success` page |
| `POST /validate` | Validates a license key |
| `POST /activate` | Activates a license key against a device fingerprint |

**Required secrets** (set via `wrangler secret put` inside `workers/license-issuer/`):

| Secret | Description |
|--------|-------------|
| `LICENSE_SIGNING_PRIVATE_KEY` | Base64-encoded 32-byte Ed25519 seed |
| `STRIPE_WEBHOOK_SECRET` | Stripe webhook signing secret (`whsec_...`) |
| `RESEND_API_KEY` | Resend API key for transactional email |

**Local testing**: `scripts/mint-test-license.mjs` generates a throwaway Ed25519 keypair and mints a license token for activating a dev build without a real purchase:

```bash
cd workers/license-issuer
node scripts/mint-test-license.mjs test@example.com
```

## Environment Variables

**Build-time** (`.env.production`, also mirrored under `wrangler.toml [vars]` for the Worker):

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_STRIPE_PAYMENT_LINK` | Stripe Payment Link; falls back to `/#pricing` |
| `NEXT_PUBLIC_DOWNLOAD_URL` | Public DMG URL (GitHub Releases or R2); drives the download button |
| `NEXT_PUBLIC_LICENSE_ISSUER_URL` | Base URL of the license-issuer Worker |
| `NEXT_PUBLIC_PH_PRODUCT_URL` / `NEXT_PUBLIC_PH_UPCOMING_URL` | Product Hunt badge link (post-launch wins if both set) |

**Main site Worker** (`wrangler.toml [vars]`):

| Variable | Description | Default |
|----------|-------------|---------|
| `PURCHASE_URL` | Stripe Payment Link; drives `/api/cta` and buy buttons | `/#pricing` |

**License-issuer Worker** (`workers/license-issuer/wrangler.toml [vars]`):

| Variable | Description |
|----------|-------------|
| `FROM_EMAIL` | Sender address for license emails |
| `SUPPORT_EMAIL` | Support address shown in license emails |

Secrets are never stored in `wrangler.toml` — set them with `wrangler secret put <NAME>`.
